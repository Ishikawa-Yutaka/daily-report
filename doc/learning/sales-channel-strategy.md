# SaaS営業チャネル戦略ガイド

## ⚠️ 重要: 2025年12月スマホ新法対応版

**このドキュメントは2025年12月18日施行の「スマートフォンソフトウェア競争促進法（スマホ新法）」に対応しています。**

**主な変更点:**
- ✅ **外部決済が正式に解禁** - Apple 15%、Google 20%の手数料
- ✅ **App Store内課金も削減** - 30% → 26%
- ✅ **完全Web課金なら0%** - Appleの課金システムを使わない場合
- ✅ **サードパーティストア解禁** - 日本でも第三者アプリストアが可能

**結論: Web直接アクセスが最も有利（手数料0%）、App Storeアプリからの誘導も可能**

## 営業チャネルの概要

マルチテナント化されたSaaSアプリは、複数の営業チャネルで顧客を獲得できます。各チャネルには特徴があり、フェーズに応じて使い分けることが重要です。

### 主要な3つのチャネル

1. **Web直接アクセス（セルフサービス）** - 手数料ゼロ、完全自動化
2. **App Store / Google Play（セルフサービス）** - 検索流入、審査あり
3. **対面営業（エンタープライズ）** - 高単価、人的コスト大

### チャネル選択の基準

| 観点 | Web直接 | App Store | 対面営業 |
|------|---------|-----------|----------|
| 手数料 | 0% | 15-30% | 0% |
| 初期コスト | 低 | 中 | 高 |
| スケーラビリティ | 高 | 高 | 低 |
| 顧客獲得単価 | 低 | 中 | 高 |
| 単価 | 中 | 中 | 高 |
| 自動化 | 完全 | 完全 | 部分的 |

## チャネル1: Web直接アクセス（推奨）

### 仕組み

```
検索エンジン / SNS / 広告
  ↓
公式Webサイトにアクセス (https://yourdomain.com)
  ↓
「無料で始める」ボタンをクリック
  ↓
サインアップフォーム
  - 会社名入力
  - 管理者情報（社員番号、氏名、パスワード）
  ↓
Organization自動作成
  ↓
即座にダッシュボードにアクセス（試用期間30日）
  ↓
[30日後]
  ↓
有料プランへのアップグレード案内
  ↓
Stripe Checkoutで課金
  ↓
継続利用
```

### メリット

#### 1. **手数料ゼロ**

```
月額5,000円 × 100社 = 50万円/月
手数料: 0円
実収入: 50万円/月（100%）
```

#### 2. **完全自動化**

- サインアップからアップグレードまで人手不要
- 24時間365日、世界中から登録可能
- スケールしやすい

#### 3. **自由度が高い**

- 価格設定を自由に変更
- デザインやメッセージを自由にカスタマイズ
- A/Bテストでコンバージョン率を最適化

#### 4. **データを完全に把握**

- Google Analytics で流入分析
- ユーザー行動を追跡
- マーケティング施策の効果測定

### デメリット

#### 1. **集客が必要**

- SEO対策（検索エンジン最適化）
- 広告費用（Google広告、SNS広告）
- コンテンツマーケティング

**対策:**

## チャネル2: App Store / Google Play

### 重要: 2025年12月スマホ新法施行

**2025年12月18日施行の「スマートフォンソフトウェア競争促進法（スマホ新法）」により、手数料体系が大幅に変更されました。**

**主な変更点:**
1. **外部決済が正式に解禁** - アプリ外のWebサイトで課金可能
2. **手数料の削減** - 従来の30%から最大26%に削減（App Store内課金）
3. **外部決済手数料の設定** - Apple 15%、Google 20%
4. **サードパーティストア解禁** - 日本でも第三者アプリストアが可能に

### 仕組み（2025年12月以降）

```
ユーザーがApp Storeで「日報」を検索
  ↓
アプリをダウンロード（無料）
  ↓
アプリ起動
  ↓
「アカウント作成」ボタンをタップ
  ↓
[選択肢1] 外部ブラウザでWebサイトに誘導（推奨）
  → Webでサインアップ・課金
  → 手数料: Apple 15% / Google 20%
  ↓
[選択肢2] App Store内課金
  → 手数料: 26%（従来の30%から削減）
  ↓
アプリでログイン
  ↓
利用開始
```

### メリット

#### 1. **検索流入**

- App Storeで「日報」「業務報告」を検索するユーザーにリーチ
- 能動的に探しているユーザー（購買意欲が高い）

#### 2. **信頼性**

- Apple/Googleの審査を通過
- ユーザーが安心してダウンロード

#### 3. **通知・プッシュ**

- モバイルアプリならではの機能
- リテンション（継続率）向上

#### 4. **外部決済が正式に認められた**（新法による）

- Appleのガイドライン違反のリスクなし
- 合法的に外部Webサイトへ誘導可能

### デメリット

#### 1. **手数料（改善されたが依然として高い）**

**2025年12月以降の手数料体系:**

| 課金方法 | Apple | Google |
|---------|-------|--------|
| **外部Web決済**（推奨） | **15%** | **20%** |
| App Store/Play内課金 | 26% | 26% |
| 小規模事業者プログラム | 10-15% | 10-15% |

**計算例:**
```
月額5,000円 × 100社 = 50万円/月

[外部Web決済の場合]
Apple手数料（15%）: 7.5万円/月
実収入: 42.5万円/月（85%）

[App Store内課金の場合]
Apple手数料（26%）: 13万円/月
実収入: 37万円/月（74%）

[従来（2025年12月以前）]
Apple手数料（30%）: 15万円/月
実収入: 35万円/月（70%）
```

**改善点:**
- 外部Web決済で15万円 → 7.5万円（50%削減）
- App Store内でも30% → 26%（4%削減）

#### 2. **審査が必要**

- 初回リリース: 1-2週間
- アップデート: 数日
- リジェクトのリスク

#### 3. **ルール制約**

- Appleのガイドラインに従う必要
- デザインや機能に制限

### App Store手数料を回避する方法

#### **方法1: アプリ内課金を使わない**（推奨）

```typescript
// React Native (Expo) の場合
import * as WebBrowser from 'expo-web-browser'

function SignupButton() {
  const handleSignup = async () => {
    // 外部ブラウザでWebサイトを開く
    await WebBrowser.openBrowserAsync('https://yourdomain.com/signup')
  }

  return (
    <Button onPress={handleSignup} title="アカウント作成（Webで）" />
  )
}
```

**Appleのルール:**
- アプリ内課金（IAP）を使わない場合、手数料なし
- ただし、アプリ内で直接的に誘導するとリジェクト
- 「詳細はWebサイトで」は OK
- 「今すぐWebで課金」は NG

**安全な表現:**
```
✅ OK: 「アカウント管理はWebサイトで行えます」
✅ OK: 「詳細は公式サイトをご覧ください」
❌ NG: 「今すぐWebで登録して50%オフ！」
❌ NG: 「App Storeより安く購入できます」
```

#### **方法2: リーダーアプリとして申請**

**リーダーアプリの定義:**
- コンテンツを「閲覧」するアプリ
- 例: Netflix、Kindle、Spotify

**日報アプリの場合:**
- 「業務報告の閲覧・記録アプリ」として申請
- 課金はWebのみ
- Appleが認める可能性あり（要審査）

**申請時の注意:**
```
- App Review Guidelineの3.1.3(a)を確認
- 「リーダーアプリ」として明記
- Webサイトへのリンクを含める
```

### 実装のポイント

#### 1. **アプリ内の導線**

```
[初回起動画面]
「日報アプリへようこそ」

[ボタン]
- 既にアカウントをお持ちの方: ログイン
- 新規の方: アカウント作成（Webサイトへ）

[タップ後]
→ Safari/Chrome で https://yourdomain.com/signup を開く
```

#### 2. **App Store最適化（ASO）**

**アプリ名:**
```
日報アプリ - チーム業務報告
（キーワード「日報」「業務報告」を含める）
```

**サブタイトル:**
```
シンプルな日報作成と共有
```

**スクリーンショット:**
- 1枚目: 日報作成画面（わかりやすく）
- 2枚目: 一覧画面
- 3枚目: 統計・分析

**説明文:**
```
日報の作成・共有をシンプルに。
モバイルからサクッと入力、チームで即座に共有。

【主な機能】
- 簡単な日報作成
- リアルタイム共有
- 検索・フィルター
- 統計分析

【こんな方におすすめ】
- 日報を効率化したい
- チームの進捗を可視化したい
- 外出先からも報告したい
```

## チャネル3: 対面営業（エンタープライズ）

### 仕組み

```
[営業活動]
テレアポ / 飛び込み / 紹介
  ↓
アポイント取得
  ↓
訪問・デモ
  - 課題ヒアリング
  - 製品デモ
  - 導入効果の提案
  ↓
見積もり提出
  ↓
契約（年間契約、カスタマイズ含む）
  ↓
導入支援
  - 初期設定
  - データ移行
  - トレーニング
  ↓
継続サポート
```

### メリット

#### 1. **高単価**

```
セルフサービス: 月額5,000円 × 12ヶ月 = 6万円/年
エンタープライズ: 100万円〜500万円/年

内訳:
- 基本料金: 50万円/年
- 導入支援: 30万円（初回のみ）
- カスタマイズ: 20万円（オプション）
- 専用サポート: 月5万円 × 12ヶ月 = 60万円
合計: 160万円/年
```

#### 2. **大企業にアプローチ可能**

- 従業員100名以上の企業
- セルフサービスでは獲得しにくい層

#### 3. **長期契約**

- 年間契約で安定収益
- 解約率が低い（導入コストが高いため）

#### 4. **カスタマイズ収益**

- 企業ごとの要望に対応
- 追加開発費を請求

### デメリット

#### 1. **人的コストが高い**

```
営業担当: 月給50万円 × 1名
営業経費: 月10万円（交通費、接待費）
導入支援: 1社あたり40時間（エンジニア工数）

1社獲得あたりのコスト: 約100万円
```

#### 2. **スケールしにくい**

- 1人の営業が対応できるのは月2-3社
- 人員を増やさないと成長しない

#### 3. **営業サイクルが長い**

```
アプローチ → 契約まで: 3-6ヶ月
```

### 提供内容（エンタープライズプラン）

#### 1. **機能面**

```
- すべての標準機能
- カスタムフィールド追加
- API連携（給与システムなど）
- SSO（シングルサインオン）
- 専用レポート作成
```

#### 2. **サポート面**

```
- 専任カスタマーサクセス担当
- 電話・メールサポート（24時間以内返信保証）
- 月1回の定例ミーティング
- トレーニングセッション（年2回）
```

#### 3. **契約面**

```
- 年間契約
- 請求書払い可能
- カスタム価格設定
- SLA（サービスレベル契約）: 99.9%稼働率保証
```

### 営業プロセス

#### 1. **リード獲得**

```
- Webからの問い合わせ（セルフサービスユーザーからのアップグレード相談）
- テレアポ（企業リストから）
- 展示会・セミナー
- パートナー紹介
```

#### 2. **初回商談（課題ヒアリング）**

```
質問例:
- 現在の日報運用方法は？（Excel、紙、メールなど）
- どんな課題がありますか？
- 何名が使用予定ですか？
- 予算感は？
- 導入時期は？
```

#### 3. **デモ・提案**

```
- 実際の画面を見せる
- 課題解決のストーリーを語る
- 導入効果を数値で示す（時間削減、コスト削減）
```

**提案例:**
```
現状: Excel で日報 → メール送信
  - 1人あたり15分/日
  - 100名 × 15分 × 20営業日 = 500時間/月

導入後: アプリで3分で完了
  - 1人あたり3分/日
  - 100名 × 3分 × 20営業日 = 100時間/月

削減効果: 400時間/月
時給換算（@3,000円）: 120万円/月の削減

導入費用: 100万円/年
投資回収期間: 1ヶ月
```

#### 4. **クロージング**

```
- 見積もり提出
- 契約書作成
- 導入スケジュール確定
```

### いつエンタープライズ営業を始めるか？

**推奨タイミング:**

```
セルフサービスで100社以上獲得後
  ↓
導入事例が豊富
  ↓
営業資料が揃う
  ↓
営業チーム組織化
```

**理由:**
- 実績がないと大企業は契約しない
- セルフサービスで成功事例を作る
- その事例を使って営業する

## その他のチャネル

```
会計ソフト（freee、マネーフォワード）と提携

「おすすめアプリ」として紹介
  ↓
紹介料: 初回契約の20%
```

**メリット:**
- 既存の顧客基盤を活用
- 信頼性が高い（提携先のお墨付き）

**デメリット:**
- 紹介料が必要
- パートナー選定・交渉が必要

### 2. **アフィリエイト**

**仕組み:**
```
ブロガー、インフルエンサーが紹介
  ↓
専用URLから登録
  ↓
紹介料: 初回契約の30%
```

**実装:**
```typescript
// アフィリエイトリンク
https://yourdomain.com/signup?ref=blogger123

// データベースに記録
await prisma.referral.create({
  data: {
    referrerId: 'blogger123',
    organizationId: newOrg.id,
    commission: 1500, // 5,000円の30%
  },
})
```

無料プラン:

- 基本機能のみ

有料プラン:

- ストレージ無制限
```

**コンバージョン率:**
```
無料ユーザー1,000名

有料ユーザー: 50名
```

### 4. **コンテンツマーケティング**

**ブログ記事:**
```
- 「効果的な日報の書き方」
- 「業務効率化のコツ10選」
- 「チームマネジメントの極意」
```

**SEO効果:**
```

## App Store手数料を最小化する方法

### 2025年12月以降の新ルール（スマホ新法）

**スマホ新法により、外部決済が正式に解禁されました。**

従来は「リーダーアプリ」などの例外を除き、外部決済への誘導は制限されていましたが、2025年12月18日以降、日本では以下が可能になりました：

1. **アプリ内で外部決済への誘導が可能**
2. **外部決済手数料が設定** - Apple 15%、Google 20%
3. **ユーザーに選択肢を提供** - App Store内課金と外部決済の両方を提示可能

### 新しい手数料体系（2025年12月以降）

| 方法 | Apple手数料 | Google手数料 | 備考 |
|------|-----------|------------|------|
| **完全Web課金**（最安） | **0%** | **0%** | アプリからWebに誘導するが、課金はApple/Google経由なし |
| **外部決済リンク** | **15%** | **20%** | アプリ内から外部決済へリンク |
| **App Store内課金** | 26% | 26% | 従来の方式（30%から削減） |
| 小規模事業者プログラム | 10-15% | 10-15% | 年間売上100万ドル以下 |

### 最も手数料を抑える方法: 完全Web課金（0%）

**重要な発見:** スマホ新法下でも、Appleの課金システムを一切使わずにWebで課金すれば**手数料0%**が可能です。

**実装方法:**

```typescript
// React Native (Expo) の場合
import * as WebBrowser from 'expo-web-browser'

function SignupButton() {
  const handleSignup = async () => {
    // 外部ブラウザでWebサイトを開く
    await WebBrowser.openBrowserAsync('https://yourdomain.com/signup')
  }

  return (
    <View>
      <Text>アカウント作成はWebサイトで行えます</Text>
      <Button onPress={handleSignup} title="Webサイトでサインアップ" />
    </View>
  )
}
```

**注意点:**
- 「詳細はWebサイトで」という案内は OK
- Appleの外部決済システムAPIを使わない場合、手数料なし
- ただし、ユーザー体験は App Store内課金より劣る

### 外部決済リンク（Apple 15% / Google 20%）

スマホ新法により、Appleが提供する「外部決済リンク」APIを使う方法も可能：

**実装方法（Appleの公式API）:**

```swift
// Swift (iOS) の場合
import StoreKit

func showExternalPurchaseOption() {
    // Appleの外部決済リンクAPI
    ExternalPurchase.canMakePayments { canMake in
        if canMake {
            ExternalPurchase.presentExternalPurchaseSheet()
        }
    }
}
```

**メリット:**
- Appleが認めた正式な方法
- ユーザー体験が良い（アプリ内でスムーズに誘導）

**デメリット:**
- 手数料15%がかかる
- Appleへの報告義務あり

### 従来のルール（参考: 2025年12月以前）

**3.1.1 In-App Purchase（アプリ内課金）**

> デジタルコンテンツやサービスをアプリ内で販売する場合、Appleのアプリ内課金を使用する必要があります。

**適用対象:**
- ゲーム内アイテム
- アプリ内の追加機能
- サブスクリプション

**適用外:**
- 物理的な商品（Amazonなど）
- 外部のサービス（Uberなど）
- **リーダーアプリ（コンテンツ閲覧アプリ）**

**3.1.3(a) リーダーアプリ**

> 「リーダー」アプリは、以前に購入または購読したコンテンツ（雑誌、新聞、書籍、オーディオ、音楽、ビデオ、専門データベースへのアクセス、VoIP、クラウドストレージ、承認されたサービス）をユーザーが閲覧できるようにするものです。リーダーアプリは、アカウント作成や管理をWebサイトで行うことができます。

**該当する例:**
- Netflix（動画閲覧）
- Kindle（書籍閲覧）
- Spotify（音楽閲覧）
- **日報アプリ（業務報告の閲覧・記録）**

### 手数料比較（2025年12月スマホ新法対応）

| チャネル | 手数料 | 実収入（月額5,000円の場合） | 備考 |
|---------|--------|--------------------------|------|
| **Web直接** | **0%** | **5,000円（100%）** | 最もお得 |
